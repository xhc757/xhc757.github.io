Github Pages 是什么#

Github Pages 是 Github 提供的免费静态内容托管服务。在 Github 新建一个名为 xhc757.github.io 的仓库，xhc757替换为你注册 GitHub 使用的用户名，稍等几分钟即可使用 xhc757.github.io 访问网站。Github Pages 站点大小不能超过1GB，因此适合用作个人博客或小型网站。

使用 Hugo 生成博客#

现在网站是空的，我们需要往仓库上传我们的网站内容。我们可以手动制作网页，但为了方便，我们通常会使用静态内容生成器。起初我尝试了基于 Python 的 Pelican，但很快因为 Pelican 没有好看的主题放弃。我转而使用 Hugo ，因为 Hugo 有一个漂亮的主题 PaperMod，而且 Hugo 安装使用非常简单。

安装 Hugo#

在 Windows 下安装 Hugo 只需要下载单个可执行文件，然后将其位置添加到 PATH 环境变量。打开 Powershell 窗口，输入 hugo version 回车，如果显示 Hugo 版本信息说明已经安装好了。

快速上手 Hugo#

在 Powershell 窗口输入 hugo new site xhc757.github.io 回车，会创建文件夹xhc757.github.io，打开文件夹会看到自动生成了很多文件和文件夹，其中 hugo.toml 是 hugo 的配置文件。

进入这个目录，输入 hugo new posts\first-blog.md 回车，这样就生成了一篇博文，我们可以用 Markdown 编辑器打开 first-blog.md 文件编写博文。注意文件头部有一些元数据，其中 draft = true 意思是这篇博客是草稿，默认情况下 Hugo 不会为其生成网页。如果我们要将此篇博文发表，应要把 draft = true 改为 draft = false。

我们还必须为博客指定一个主题，下载一个 hugo theme 比如 PaperMod，放到 themes 文件下，然后编辑hugo.toml，输入 theme = ‘PaperMod’。

接下来输入 hugo，就会生成网页，输出在 public 文件夹。这个文件夹不会自动清空，如果有需要，我们可以在生成前手动清空 public 文件夹。

然后输入 hugo server 回车，在浏览器打开 http://localhost:1313/ 就可以预览博客了。

推送到 Github#

使用 hugo server 命令时生成的网页 url 前缀是基于 localhost 的，方便我们在本地预览。所以在推送前务必使用 hugo 重新生成网页，然后再使用 Sourcetree 把 public 文件夹下面的所有文件推送到 xhc757.github.io 仓库
